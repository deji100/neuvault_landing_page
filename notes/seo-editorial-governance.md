# NeuVault Editorial Governance

Updated: 2026-04-07

## Rule set

- One primary intent per page.
- One product workflow page per commercial intent.
- One support guide only when the query is clearly distinct from the product page.
- Do not publish near-duplicate pages for adjacent keyword variations.
- New pages must answer a question the current site does not already answer well.

## Keep / merge / delay / discard

### Keep next
These are distinct enough to justify a future page if needed.
- scanner app vs document vault
- best app for scanning and organizing family documents
- best app to organize receipts, contracts, and freelance documents
- how to organize travel documents, visas, and IDs digitally
- best way to manage medical and insurance documents privately
- how to export notes and documents without vendor lock-in

### Merge into existing pages or guides
Do not create standalone pages for these unless the site evolves significantly.
- app to scan and store important documents securely
  Merge into: `/scan-organization`
- document scanning and organizing software for personal use
  Merge into: `/scan-organization`
- securely store scanned documents without using random folders
  Merge into: `/scan-organization` or its guide
- never miss a document renewal date again
  Merge into: `/document-reminder`
- best way to set reminders for important documents
  Merge into: `/document-reminder`
- best app to retrieve important documents in seconds
  Merge into: `/document-retrieval`
- how to create a searchable personal document vault
  Merge into: `/document-retrieval`
- private backup for personal records, IDs, and certificates
  Merge into: `/secure-document-backup`
- securely export your notes, scans, and records
  Merge into: `/notes-export`

### Delay
Useful later, but not before current pages show movement.
- best document organizer app for students
- how families can keep important documents organized in one place
- best app for organizing business and personal documents separately
- reminder app for bills, contracts, and official paperwork
- track renewal dates for insurance, licenses, and IDs
- cloud convenience without giving up control of your documents

### Discard unless the product positioning changes
These are most likely to produce weak, overlapping, or low-value pages if published now.
- multiple small variants that differ only by wording, not by intent
- separate pages for every reminder synonym
- separate pages for every scan/store/organize synonym
- pages built only to repeat the same privacy/local-first talking points without a new use case

## Publishing checklist

Before publishing a new page, confirm:
- What query does this page own that no existing page already owns?
- What user problem does it solve better than the current site?
- What parent workflow page will it support?
- What internal links point into it?
- What internal links point back up to the parent page?
- What evidence, examples, or practical value make it non-commodity?

If those answers are weak, do not publish the page.
